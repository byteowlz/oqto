Implementation Plan
===================

1. Backend (4 new endpoints):

   GET  /api/admin/eavs/status
     -> { configured: bool, providers: [{ name, type, healthy, models_count }] }
     -> Called on login and by settings page

   POST /api/admin/eavs/providers
     -> Body: { name, type, api_key, base_url?, test_model? }
     -> Writes to eavs config.toml + env file
     -> Restarts eavs service (systemctl or signal)
     -> Returns: { provider: { name, type, healthy } }

   POST /api/admin/eavs/test
     -> Body: { provider_name } OR { type, api_key, base_url? }
     -> Calls eavs health/test endpoint
     -> Returns: { success, error?, latency_ms, models_found }

   POST /api/admin/eavs/sync
     -> Regenerates models.json for all users
     -> Returns: { models_count, providers_count }

   DELETE /api/admin/eavs/providers/{name}
     -> Removes provider from config, restarts eavs

2. Frontend Components (shared between wizard + settings):

   ProviderForm         -- API key input, type selector, base_url for custom
   TestConnectionButton -- Calls POST /eavs/test, shows spinner/result
   ProviderCard         -- Status badge, model count, edit/test/remove actions
   ProviderGrid         -- Card grid layout with "+ Custom" button

   EavsSetupWizard      -- Full-screen stepper (first-run only)
     Step 1: Select providers (checkboxes: Anthropic, OpenAI, Google, Custom)
     Step 2: Enter API keys (one per selected provider, with inline test)
     Step 3: Review + confirm (shows all green checks)
     Step 4: Syncing models... (calls POST /eavs/sync, shows progress)

   SettingsProvidersPage -- Permanent settings page using ProviderGrid
     /settings/providers route
     Add/edit/remove/test providers at any time

3. App Shell Integration:

   On login:
     GET /api/admin/eavs/status
     if (!configured && user.role === 'admin')
       -> Show EavsSetupWizard (full-screen, blocks app)
     if (!configured && user.role !== 'admin')
       -> Show banner: "Waiting for admin to configure LLM providers"
     if (configured)
       -> Normal app shell

   Session creation:
     Disable "New Session" button if eavs not configured
     Tooltip: "No LLM providers configured"

4. Security:

   All /api/admin/eavs/* endpoints require admin role
   API keys are write-only (never returned in GET responses)
   Test endpoint accepts ephemeral keys (not persisted until confirmed)
   eavs restart is rate-limited (max 1 per 10s)

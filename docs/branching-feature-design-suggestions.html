Branching UX: 10 Design Suggestions

Goal: Make branching/forking first-class in the UI (sidebar + optional overlay) without introducing Pi-specific backend coupling.

Canonical graph model (harness-agnostic) Sidebar visibility Fast switch + safe recovery Scales from simple to power users
01Best for MVP

Tree Sidebar Mode

Add a List/Tree toggle to sessions sidebar. Branches render as nested nodes under origin.

OUTATIME ├─ main │ └─ branch-a (active) └─ branch-b
  • Low implementation risk
  • Clear lineage at a glance
  • No modal required
SidebarLow complexity
02Power users

Graph Overlay (DAG)

A dedicated overlay showing nodes and edges. Click node to preview and switch.

[main]──►[variant-1] └────►[variant-2]──►[variant-2b]
  • Excellent for many forks
  • Supports filters and diff hints
  • Best debugging/recovery UX
OverlayHigh clarity
03Conversation-first

Inline Fork Markers

Show "Forked from this message" badges directly inside transcript at fork points.

Message #84 [Fork point] 2 children [Open branch A] [Open branch B]
  • Contextual and discoverable
  • Minimal navigation jumps
  • Good for small trees
Chat timelineContextual
04Quick iteration

Branch Tabs per Session

Represent branch heads as tabs above chat. Tabs map to branch nodes.

[main] [image-fix] [prompt-v3*]
  • Fast switching
  • Familiar browser-like UX
  • Works with small branch counts
TabsFast switch
05Structured workflows

Branch Board (Kanban)

Visual board grouping branches by status: Exploring, Validating, Kept, Archived.

Exploring | Validating | Kept branch-a branch-b main
  • Great for teams
  • Makes branch intent explicit
  • Improves cleanup discipline
WorkflowTeam-friendly
06Diff-driven

Compare Mode

Pick two branch nodes and show response/tool diff summary before switching.

Compare: main ↔ prompt-v3 + 2 tool calls - 1 retry + Better final answer length
  • Reduces guesswork
  • Supports decision making
  • Pairs well with graph overlay
DiffAnalysis
07Scalable nav

Minimap + Focus Lens

Tiny branch minimap in sidebar with viewport highlight; main area focuses selected subtree.

[mini graph] focus: branch-2 subtree
  • Handles deep trees
  • Maintains orientation
  • Compact UI footprint
NavigationLarge trees
08Safety-first

Pinned Branch Heads

Allow pinning branch heads. Sidebar shows pinned section independent of chronology.

Pinned • release-candidate • best-image-style
  • Prevents losing valuable branches
  • Good for long sessions
  • Simple to implement
PinningRetention
09Guided UX

Fork Wizard

Small guided flow: choose source message, name branch, optional goal label, create.

Step 1: pick message Step 2: branch name Step 3: optional intent tag
  • Reduces accidental forks
  • Improves branch naming quality
  • Good for new users
OnboardingGuided
10Recommended mix

Hybrid: Tree + Overlay

Default tree sidebar for daily use, plus optional graph overlay for complex branch management.

Default: Tree Advanced: Open Graph Overlay Actions: fork, switch, compare, archive
  • Balanced complexity
  • Fast path + power path
  • Best long-term architecture fit
BalancedPhased rollout
Suggested rollout: Phase 1: Tree Sidebar (01) + Inline Markers (03). Phase 2: Compare Mode (06). Phase 3: Graph Overlay (02) with pin/archive controls (08).
